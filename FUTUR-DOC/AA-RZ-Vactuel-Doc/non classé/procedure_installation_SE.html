Procédure d'Installation SE — RobotRZ

Installation complète depuis zéro — Téléphone Doro (Android) — Version 2026-04-28

Prérequis

  • Téléphone Android (Doro) avec accès au Play Store
  • Connexion WiFi (même réseau que le RPI broker)
  • Accès SSH depuis un PC (optionnel mais recommandé)
  • Dépôt Git : https://github.com/lugvincent/RobotRZ

Étape 1 — Installation des applications Android

1
Installer Termux depuis F-Droid (pas le Play Store — version maintenue)
2
Installer Termux:API depuis F-Droid (même source que Termux)
3
Installer Termux:Tasker depuis F-Droid
4
Installer Tasker depuis le Play Store (version payante recommandée)
5
Installer IP Webcam Pro (com.pas.webcam.pro) depuis le Play Store — Thyoni Tech
⚠️Termux et Termux:API doivent venir de la même source (F-Droid). Mélanger Play Store et F-Droid empêche la communication entre les applis.

Étape 2 — Configuration Termux

# Accorder les permissions de stockage
termux-setup-storage

# Mettre à jour les packages
pkg update && pkg upgrade -y

# Installer les dépendances essentielles
pkg install -y git jq openssh curl mosquitto python

# Installer les packages Python
pip install paho-mqtt pyserial --break-system-packages

# Activer SSH (port 8022 par défaut)
sshd
⚠️Le port SSH de Termux est 8022 (pas 22). Connexion : ssh rzServeur@<IP_DORO> -p 8022

Étape 3 — Clone du dépôt

# Cloner le dépôt
cd ~
git clone https://github.com/lugvincent/RobotRZ robotRZ-smartSE

# Configurer l'identité Git
git config --global user.name "lugvincent"
git config --global user.email "vincentphcom@gmail.com"

# Configurer le token GitHub (remplacer TOKEN par votre PAT)
git remote set-url origin https://lugvincent:TOKEN@github.com/lugvincent/RobotRZ

Étape 4 — Création de keys.json (OBLIGATOIRE — non versionné)

🔴keys.json n'est pas dans Git pour des raisons de sécurité. Il doit être créé manuellement à chaque nouvelle installation.
mkdir -p ~/robotRZ-smartSE/termux/config

nano ~/robotRZ-smartSE/termux/config/keys.json

Contenu du fichier :

{
    "GEMINI_API_KEY": "VOTRE_CLE_GEMINI",
    "MQTT_USER":      "rpibrokerUser",
    "MQTT_PASS":      "VOTRE_MOT_DE_PASSE_MQTT",
    "MQTT_HOST":      "192.168.1.40"
}

Étape 5 — Génération des fichiers de config

# Exécuter check_config.sh (génère tous les *_runtime.json)
cd ~/robotRZ-smartSE/termux/config
bash check_config.sh

Étape 6 — Démarrage du système

# Démarrage complet (lance tous les managers actifs)
bash ~/robotRZ-smartSE/termux/scripts/core/rz_start.sh

# Vérifier l'état des processus
bash ~/robotRZ-smartSE/termux/scripts/core/rz_start.sh --status

# Arrêt propre
bash ~/robotRZ-smartSE/termux/scripts/core/rz_start.sh --stop

Managers démarrés automatiquement selon courant_init.json :

  • rz_vpiv_parser.sh — toujours
  • rz_state-manager.sh — toujours
  • rz_sys_monitor.sh — si modeSys != OFF
  • rz_gyro_manager.sh — si modeGyro != OFF
  • rz_mg_manager.sh — si modeMg != OFF
  • rz_stt_manager.sh — si modeSTT != OFF

Étape 7 — Vérification SSH depuis le PC

# Connexion SSH standard (réseau local)
ssh rzServeur@192.168.1.12 -p 8022

# Via NAT box (depuis internet)
ssh rzServeur@192.168.1.12 -p 5800

# Vérifier les processus actifs
ps | grep -E "rz_|mosquitto|python"

# Vérifier la connexion MQTT
mosquitto_pub -h 192.168.1.40 -p 1883 \
    -u "rpibrokerUser" -P "VOTRE_MOT_DE_PASSE" \
    -t "SE/statut" -m "test" -q 0
