3. Tâche prioritaire — RZ_CamStart / RZ_CamStop

Ces deux tâches sont bloquantes pour la validation complète de CamSE.

RZ_CamStart

1
Action Tasker : Launch App → com.pas.webcam.pro
2
Action Tasker : Wait → 2 secondes (laisser l'appli démarrer)
3
Action IP Webcam Pro (plugin natif Tasker Pro) : Start Server
4
rz_camera_manager.sh prend le relai — attend port 8080 et publie StreamURL

RZ_CamStop

1
Action IP Webcam Pro (plugin) : Stop Server
2
Action Tasker : Kill App → com.pas.webcam.pro
⚠️IP Webcam Pro version payante requise pour l'intégration Tasker native. La version gratuite ne supporte pas le plugin Tasker.

4. Déclenchement depuis Termux

Le script rz_tasker_bridge.sh reçoit le nom de la tâche et les paramètres :

# Appel depuis rz_vpiv_parser.sh via fifo_tasker_in
# Format de la trame dans la FIFO :
$A:Appli:launch:*:Baby#        → tâche RZ_Baby
$A:CamSE:modeCam:rear:stream#  → tâche RZ_CamStart
$A:Appli:expr:*:sourire#       → tâche RZ_Expression

# rz_tasker_bridge.sh extrait le nom et appelle :
termux-tasker NOM_TACHE PARAMETRE
⚠️Les scripts appelés par Termux:Tasker doivent être dans ~/.termux/tasker/. Vérifier ce répertoire lors de la configuration.
