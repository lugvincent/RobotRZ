Unités et codage

Valeurs brutes

Entiers = rad/s × 100

Ex: [120, -45, 300]

→ Gx=1.20 rad/s, Gy=-0.45, Gz=3.00

Formats supportés

Tableau : [x, y, z] ← format réel SE

Objet : {"x":120,"y":-45,"z":300}

Les deux sont parsés par Node-RED

Mag — Magnétomètre MMC3630KJ Validé
termux-sensor · degrés × 10 · mode DATAMgCont

État au 28 avril 2026

  • Capteur MMC3630KJ détecté par termux-sensor
  • rz_mg_manager.sh opérationnel en mode DATAMgCont
  • Trames $F:Mag:DataContMg:*:<cap_deg10># reçues dans Node-RED
  • Parsing Node-RED validé (fn_extract_mag_v2)
  • Dashboard Diag-SE affiche cap en degrés et point cardinal
  • mag_runtime.json généré par check_config.sh
  • Valeurs = 0 fréquentes quand téléphone à plat (normal — axe Z dominant)
  • Relai compass (cap géo → Arduino) non encore implémenté
  • Mode DATAMgMoy non testé
  • Mode DATAGeoCont / DATAGeoMoy non testés
  • Calibration magnétique non réalisée

Architecture

MMC3630KJ
→ termux-sensor →
rz_mg_manager.sh
→ fifo_termux_in →
rz_vpiv_parser.sh
→ MQTT SE/statut →
SP Node-RED

Configuration (courant_init.json)

{
  "mag": {
    "modeMg": "DATAMgCont",
    "paraMg": {
      "freqCont":    5,       // Hz
      "freqMoy":     1,       // Hz
      "nbValPourMoy": 5,
      "seuilMesure": { "x": 50, "y": 50, "z": 50 }  // µT×100
    }
  }
}

VPIV (Table A)

DirectionTrameDescription
SP→SE$V:Mag:modeMg:*:DATAMgCont#Cap magnétique continu
SP→SE$V:Mag:modeMg:*:DATAGeoCont#Cap géographique continu
SP→SE$V:Mag:modeMg:*:DATABrutCont#Champ brut continu [x,y,z]
SP→SE$V:Mag:modeMg:*:OFF#Arrêt magnétomètre
SE→SP$F:Mag:DataContMg:*:1850#Cap magnétique (degrés×10)
SE→SP$F:Mag:DataContGeo:*:1830#Cap géographique (degrés×10)
SE→SP$F:Mag:dataContBrut:*:[x,y,z]#Champ brut (µT×100)
SE→SP$I:Mag:modeMg:*:DATAMgCont#ACK mode

Unités et codage

Cap magnétique / géographique

Entier = degrés × 10

Ex: 1850 → 185,0° (Sud)

Ex: 3450 → 345,0° (Nord-Nord-Ouest)

Plage : 0 à 3599 (0,0° à 359,9°)

Champ brut

Tableau [x, y, z] en µT × 100

Ex: [1234, -567, 890]

→ Bx=12.34µT, By=-5.67µT, Bz=8.90µT

⚠️ Valeurs = 0 si téléphone à plat (normal)

⚠️ Orientation requise : le magnétomètre donne des valeurs fiables uniquement quand le téléphone est tenu verticalement. En position horizontale (à plat), les axes X et Y sont sous le seuil de mesure (50 µT×100) et la valeur retournée est 0. C'est le mode d'utilisation normal sur le robot (téléphone monté verticalement).
