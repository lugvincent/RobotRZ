2. Qu'est-ce qu'une FIFO ?

FIFO = First In First Out - un "tuyau nomme" (pipe nomme) entre deux processus sur le meme systeme. C'est un fichier special reconnaissable au p devant ses permissions : prw-rw-rw-. Un processus ecrit dedans, un autre lit a l'autre bout - dans l'ordre d'arrivee.

manager (ex: sys_monitor) ecrit une trame
--> fifo_termux_in -->
rz_vpiv_parser.sh lit et publie MQTT
--> MQTT -->
SP (Node-RED)
Regle fondamentale : les managers SE n'ecrivent JAMAIS directement avec mosquitto_pub. Tout passe par la FIFO - rz_vpiv_parser.sh - MQTT.

3. Demarrage minimal pour tester

# 1. Tuer les eventuels processus restants
kill $(pgrep -f "rz_vpiv_parser") 2>/dev/null
kill $(pgrep -f "rz_sys_monitor") 2>/dev/null

# 2. Lancer le parser en arriere-plan
bash ~/robotRZ-smartSE/termux/scripts/core/rz_vpiv_parser.sh &

# 3. Attendre que le parser soit pret
sleep 2

# 4. Lancer le monitor systeme
bash ~/robotRZ-smartSE/termux/scripts/sensors/sys/rz_sys_monitor.sh
Resultat attendu : des lignes FIFO-MQTT s'affichent toutes les 10 secondes avec CPU, thermique, stockage, RAM, batterie.

8. Points de vigilance

  • error_handler.sh - utilise ancienne URL DuckDNS sans credentials. A corriger avec keys.json.
  • mqtt_bridge.py - utilise ancienne URL DuckDNS sans credentials. A corriger avec keys.json.
  • rz_state-manager.sh - ecoute MQTT directement sur SE/commande en parallele du parser. Double abonnement possible. A clarifier architecturalement.
  • fr.dict - 17 mots sans phonetique : rz, gps, leds, tasker, reset, snapshot, vas-y, reveille-toi, active-toi, desactive, eteins, localise, reinitialise, d'ou, l'alerte, l'anneau, l'urgence.
